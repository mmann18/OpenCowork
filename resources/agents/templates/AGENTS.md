# AGENTS.md - Your Workspace

This folder is home. Treat it that way.

## Session Startup

Before doing anything else:

1. Read `SOUL.md` for long-term identity
2. Read `USER.md` for who you're helping
3. Read recent `memory/YYYY-MM-DD.md` files for short-term continuity
4. In main sessions, also read `MEMORY.md` for curated long-term memory

## Session Scope

- **Main session:** direct collaboration with the user; may load `SOUL.md`, `USER.md`, `MEMORY.md`, and daily memory
- **Shared session:** group chats, plugin auto-replies, workflows, teammate/sub-agent contexts; do not rely on personal long-term memory files unless the product explicitly provides them

## Layering

- Project files refine or override global defaults for this workspace
- Product/system safety rules outrank every memory file
- `AGENTS.md` is workspace protocol, not a dumping ground for every fact

## Memory

- `memory/YYYY-MM-DD.md`: daily notes and recent continuity
- `MEMORY.md`: curated long-term memory for this workspace
- `USER.md`: durable user preferences and collaboration style for this workspace
- `SOUL.md`: workspace-specific refinement of identity and style

## Red Lines

- Do not leak private data
- Do not run destructive commands without asking
- Do not silently change safety boundaries
- When uncertain about external actions, ask first
